# Interpreter in Go

## ステップ
- 字句解析器(Lexer)
  - ソースを意味のある最小単位の文字列(=トークン)に分解する作業 = トークナイズ
- 構文解析器(Parser)
  - トークンを構造化データに変換
  - データ構造の多くは構文木やAST(抽象構文木)が多い
  - パーサジェネレータ(bison, yacc, ANTLR)は今回使わない
  - コンピュータサイエンスで最もよく研究されている分野
  - 以下2つの手法がある
    - トップダウン構文解析
      - 再帰下降構文解析
        - Pratt構文解析 <==こちらを今回利用
      - アーリー法
      - 予測的構文解析
    - ボトムアップ構文解析
- 評価器(Evaluator)
